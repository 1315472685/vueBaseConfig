Views 管理方案

在项目研发中，常有多人同时研发或参与的环节，为避免在增删改查、修复冲突上花费过长时间，我们重新整理一下项目结构目录的分配和建议

模板示例：

views
│─ type                     分类
│     └─ module               模块
│           │─ .vue           对应vue
│           │─ .less          对应样式
│           └─ images         对应图片

根据以上模板示例，我们将项目中常见视图分为 官网、后台、异常三大分类，再根据分类对模块进行划分。

项目实例：

views
│─ admin(后台)
│    │─ user           用户管理
│    │─ .vue           对应vue
│    │─ .less          对应样式
│    └─ images         对应图片
│─ index(官网)
│    │─ home           首页
│    │─ .vue           对应vue
│    │─ .less          对应样式
│    └─ images         对应图片
└─ error(异常)
      │─ 404            404页
      │─ .vue           对应vue
      │─ .less          对应样式
      └─ images         对应图片

这是简单的项目实例，在实际项目中可根据分类以及模块内容进行拓展，同时可与Router 管理方案搭配，形成同一套管理思维
