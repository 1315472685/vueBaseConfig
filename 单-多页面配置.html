单/多页面配置

通过配置 config/index.js 中的 moduleName 来选择单页面应用还是多页面应用。单页面、多页面可随时互相切换，两者互不影响，路由须分开设置。

多页面和单页面唯一的不同就在于，单页面只提供一个入口(main.js)一个页面模板(index.html)，而多页面则需提供多个入口及对应的页面模板。

先来看看 vue 项目基本结构：

src | └─views │ └─ admin （管理员页面） | └─ ... | └─ entry-admin.js
（入口文件） | └─ admin.html （入口模板） | └─ login （登录页面） | └─ ... | └─
login-admin.js （入口文件） | └─ login.html （入口模板） | └─ ... └─ app.vue └─
main.js (单页面入口) | index.html (单页面入口模板)

单页面

场景需要：集成页面转场动画和模块复用的优势，常用于 pc 端官网和系统管理类型网站

  • 默认为单页面（即 moduleName 为 null）

  • 默认入口文件 src/main.js

  • 默认页面模板 index.html

多页面

场景需要：同一个项目多个 H5 应用或模块；把多个应用做成一个单页体积会很大，而且难于维护；因为各页之间相对独立，少了各种变量污染的担忧

使用现有脚手架，配置多页面只需完成以下操作步骤即可：

  • 1、设置多页面，须配置 moduleName 为项目页面所在文件夹，如设置为 views

  • 2、在每个页面添加以 entry- 为前缀的入口 js 文件（为了避开项目已有 js 文件），还要添加与页面文件夹同名的 html模板

多页面使用示例，入口和模板基本和单页面配置一样

  • 代码示例（如 views/admin 文件下）

    // 例：entry-admin.js

import Vue from 'vue'
import App from './index/index'
// import ...
new Vue({
  el: '#admin',
  components: {
    App
  },
  // ...,
  template: '<App/>'
})

    <!-- 例：admin.html -->

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width,initial-scale=1.0" />
    <link
      rel="shortcut icon"
      href="static/faviconsg.ico"
      type="image/x-icon"
    />
    <title>admin</title>
  </head>
  <body>
    <div id="admin"></div>
    <!-- built files will be auto injected -->
  </body>
</html>

  • 多页面在访问页面时不同于单页面，需要指定到对应模板名路径，例如 localhost:8080/views/admin.html
